一、Codex 是什么

Codex 是一个 AI 编程助手，但它和普通的代码补全工具不一样。你可以把它想象成一个坐在你旁边、能看懂你整个项目的程序员搭档。

你只要告诉它你想做什么——比如"帮我写一个贪吃蛇游戏"——它就会自己读你的文件、写代码、运行命令，甚至帮你调试。

💡 简单说：你在聊天框里说要什么，Codex 帮你写出来。

Codex 由 OpenAI 开发，底层默认使用 GPT 系列模型。对于国内用户，我们可以通过配置改用 DeepSeek 等国产模型，速度和费用都更友好。

二、下载安装 Codex

下面是最简单的安装方式，尽量避开命令行操作。

第一步：安装 Node.js

Codex 需要 Node.js 运行环境。把它当成一个"底座"就好：

1

打开 nodejs.org，下载 LTS 版本的安装包（左侧那个）

2

双击安装包，一路点「Next」安装完成

3

打开终端（Windows 按 Win+R，输入 cmd 回车），输入以下命令验证：

node --version

如果显示 v18.x.x 之类的版本号，说明安装成功。

第二步：安装 Codex

1

在终端中输入以下命令（复制粘贴即可）：

npm install -g @openai/codex
2

安装完成后，验证：

codex --version

三、网络配置

Codex 默认需要访问 OpenAI 的服务器。国内用户有以下几种选择：

先装 Watt Toolkit（加速 GitHub）

Watt Toolkit 只能加速 GitHub 访问，不能加速 OpenAI API。

1

打开 steampp.net，下载 Watt Toolkit（原名 Steam++）

2

安装后打开，切换到「网络加速」标签

3

勾选 GitHub，点击「一键加速」

✅ 完成！GitHub 访问快了，可用来下载 ccswitch 等工具。

方案 A：通过 GPT 中转 API

国内有很多服务商提供 OpenAI API 的中转服务。你付费买额度，他们帮你转发到 OpenAI。

💡 在中转服务商那拿到中转地址和 Key 后，用 codex config set openai-base-url 你的地址 配置即可。

方案 B：直接使用 DeepSeek API（推荐）

DeepSeek 是国产大模型，国内直连，不需任何代理。价格是 GPT 的几十分之一。

具体操作见下一节。

四、接入 DeepSeek API

DeepSeek 是国内的大模型服务商，API 在国内可直接访问，无需任何代理工具。

获取 API Key

1

打开 platform.deepseek.com，注册账号

2

进入「API Keys」页面，点击「创建 API Key」

3

复制生成的 Key（以 sk- 开头），保存好

💸 DeepSeek 的价格大约是 GPT 的 1/20，非常便宜。首次注册通常有免费额度。

配置 Codex 使用 DeepSeek

在终端中运行以下命令，告诉 Codex 使用 DeepSeek 的接口：

codex config set model-provider deepseek
codex config set deepseek-api-key sk-你的Key

或者直接编辑配置文件 ~/.codex/config.json：

{ "model": { "provider": "deepseek", "deepseek": { "api_key": "sk-你的DeepSeekAPIKey", "model": "deepseek-chat" } } }

✅ 配置完成后，Codex 就会通过 DeepSeek 在国内直连使用，又快又便宜。

五、基础配置

Codex 开箱即用，但了解一下这些配置会让体验更好：

📁 工作目录

Codex 默认在你打开终端的目录下工作。先 cd 到你的项目文件夹再启动。

🌐 代理设置

如果走代理，设置环境变量：set HTTPS_PROXY=http://127.0.0.1:1080

🔑 API Key 管理

可配置多组 Key 轮流使用，或设置费用上限防止超支。

🤖 自定义指令

在项目根目录放一个 .codexrules 文件，写入你想要的默认行为。

六、第一次使用

安装配置完成后，打开终端，cd 到你的项目目录，输入：

codex

终端会进入对话模式，你就可以直接告诉 Codex 你想做什么了。比如：

💬 "帮我创建一个贪吃蛇游戏，HTML 单文件，20×20 网格，支持触屏操作"

Codex 会理解你的需求，自动写代码、测试、修改，直到完成。

七、部署与国内访问

你的小项目做好后，需要部署到线上才能分享给别人。这里有两种方案：

方案 A：Vercel（国际用户推荐）

本项目的默认部署平台。速度快、配置简单，但国内部分地区访问不稳定。

方案 B：GitHub Pages（国内用户推荐）

GitHub 提供的免费静态托管，国内访问比 Vercel 更稳定。

1

打开你的 GitHub 仓库 Settings → Pages

2

在「Source」处选择 GitHub Actions

3

推送到 main 分支后，仓库中已配好的 .github/workflows/deploy-pages.yml 会自动构建并部署

4

部署完成后访问 https://yanxou.github.io/Code/ 即可

💡 GitHub Pages 国内直连，不需要任何工具。访问速度取决于当地网络，通常比 Vercel 流畅。
